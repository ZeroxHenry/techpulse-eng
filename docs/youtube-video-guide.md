# YouTube 영상 제작 가이드 — 0xH (헥세이치)

> **0xH** 채널 영상 제작을 위한 내부 참고 문서
> 로봇 엔지니어 Henry의 **강의형 기술 영상** 워크플로우

---

## 브랜드

**채널명:** 0xH (헥세이치)
**의미:** 0x(hex 접두사, 엔지니어/개발자) + H(Henry)
**블로그:** StackPulse (techblips.com)

---

## 영상 컨셉

**형식:** 강의형 (PPT/슬라이드 기반)
**구조:** 문제 → 개선/해결 → 결과
**출연:** VTuber 아바타 (초반), 추후 실물 전환 가능
**톤:** 엔지니어가 엔지니어에게 설명하는 느낌

```
블로그 글 작성 → 대본(KO) 작성 → 영어 번역(EN) → PPT 제작 → 녹화 → 편집+자막 → 업로드 → StackPulse 연동
```

하나의 스터디 세션 = **YouTube 영상 1개 + StackPulse 블로그 글 1개**

## 1. 스크립트/대본 템플릿

### 핵심 공식: 문제 → 개선 → 결과

모든 영상은 이 3단계를 반복하며 진행한다.

```
"이런 문제가 있었습니다"
    ↓
"이렇게 개선해서 작업했습니다"
    ↓
"결과는 이렇게 됐습니다"
```

### 기본 구조 (총 15-20분)

#### 인트로 (30초-1분)
```
[아바타 + 타이틀 슬라이드]
"안녕하세요, 로봇 엔지니어 Henry입니다.
오늘은 [주제]에서 겪었던 문제와 해결 과정을 공유합니다.
[한 줄 훅]"
```

**훅 예시:**
- "이거 연결만 하면 된다더니, 한 달 걸렸습니다."
- "공식 문서에는 3줄이면 된다는데, 현실은 달랐습니다."
- "이 에러 하나 때문에 일주일을 날렸습니다."

#### 목차/개요 슬라이드 (30초)
```
[PPT - 목차 슬라이드]
"오늘 다룰 내용입니다.
1. [문제 상황]
2. [시도한 방법들]
3. [최종 해결책과 결과]"
```

#### 문제 설명 (2-3분)
```
[PPT - 문제 상황 슬라이드 + 다이어그램]
- 어떤 프로젝트에서 이 문제를 만났는지
- 에러 메시지, 증상을 슬라이드에 캡처
- 구성도/아키텍처 다이어그램으로 어디가 문제인지 표시
```

#### 개선/해결 과정 (8-12분) ← 핵심 파트
```
[PPT + 화면녹화 교차]

시도 1: "처음에는 이렇게 해봤습니다"
  → [슬라이드: 방법 설명]
  → [화면녹화: 실제 실행]
  → [슬라이드: 결과 - 실패/부분 성공]

시도 2: "그래서 이렇게 바꿨습니다"
  → [슬라이드: 개선 포인트 하이라이트]
  → [화면녹화: 실행]
  → [슬라이드: 결과]

최종: "결국 이렇게 해서 됐습니다"
  → [슬라이드: 최종 구성]
  → [화면녹화: 동작 시연]
```

#### 결과 정리 (2-3분)
```
[PPT - 요약 슬라이드]
- Before/After 비교 슬라이드
- 핵심 포인트 3가지 이내 (번호 매기기)
- "이걸 통해 배운 것" 한 줄
- (선택) 정리 표/다이어그램
```

#### 아웃트로 (30초)
```
[아바타 + 엔딩 슬라이드]
"전체 코드와 상세 정리는 StackPulse 블로그에 있습니다.
링크는 설명란에요.
다음에는 [다음 주제]를 다루겠습니다."
```

### 스크립트 예시: ZED X Mini + Jetson

```
[인트로 - 아바타 + 타이틀]
"ZED X Mini, Jetson에 연결만 하면 된다고 합니다.
저는 한 달 걸렸습니다."

[목차 슬라이드]
"오늘 내용:
1. GMSL2가 뭔지, 왜 USB처럼 안 되는지
2. 보드 3개, JetPack 4종 조합 테스트
3. 최종 동작하는 스택"

[문제 - PPT: GMSL2 아키텍처 다이어그램]
"이 카메라는 USB가 아니라 GMSL2입니다.
(다이어그램) 여기서 여기까지 맞아야 동작합니다."

[시도1 - PPT + 화면녹화]
"reComputer로 먼저 시도했습니다."
(화면녹화: 설치 과정 → 실패)
(슬라이드: 실패 원인 정리)

[시도2 - PPT + 화면녹화]
"Waveshare 보드로 바꿨습니다."
(화면녹화: 설치 → 부분 성공)
(슬라이드: JetPack 버전 이슈 설명)

[최종 - PPT: 성공 스택 표]
"이 조합으로 됐습니다."
| Board | Waveshare Orin NX (22-pin CSI) |
| JetPack | 6.2.1 |
| ZED SDK | 5.2.1 |

[결과 시연 - 화면녹화]
(zed-explorer 실행 → depth 영상 나오는 화면)

[아웃트로 - 아바타]
"전체 트러블슈팅은 블로그에 정리했습니다.
다음에는 depth map 추출을 다루겠습니다."
```

## 2. PPT/슬라이드 구성

### 슬라이드 종류

| 슬라이드 타입 | 용도 | 빈도 |
|--------------|------|------|
| **타이틀** | 영상 제목 + 한 줄 요약 | 1장 |
| **목차** | 오늘 다룰 내용 3-4개 | 1장 |
| **문제 상황** | 에러 캡처, 증상 설명 | 2-3장 |
| **다이어그램** | 아키텍처, 연결도, 흐름도 | 2-4장 |
| **코드/명령어** | 핵심 코드, 터미널 명령 | 3-5장 |
| **비교표** | Before/After, 버전 호환성 | 1-2장 |
| **요약** | 핵심 포인트 정리 | 1장 |
| **엔딩** | 블로그 링크, 다음 예고 | 1장 |

### 슬라이드 디자인 규칙

**색상:** StackPulse 브랜드 컬러 활용
- 배경: 다크 (#1a1a2e or #0f0f23)
- 주요 강조: Cyber Teal (#0d9488)
- 텍스트: 밝은 회색 (#e0e0e0)
- 에러/경고: 빨간색 (#ef4444)
- 성공: 녹색 (#22c55e)

**폰트:**
- 제목: 굵게, 28-36pt
- 본문: 20-24pt (너무 작으면 모바일에서 안 보임)
- 코드: 모노스페이스 (Fira Code, JetBrains Mono), 18-22pt

**레이아웃 원칙:**
- 한 슬라이드에 한 가지 내용만
- 글자 수: 슬라이드당 최대 5줄
- 다이어그램은 단순하게 (박스 + 화살표)
- 코드는 핵심 부분만 하이라이트

### PPT 제작 도구

| 도구 | 장점 | 추천 용도 |
|------|------|-----------|
| **Google Slides** | 무료, 협업, 어디서든 접근 | 기본 추천 |
| **Figma** | 디자인 자유도 높음 | 다이어그램 많은 영상 |
| **Canva** | 템플릿 풍부, 빠른 제작 | 빠르게 만들고 싶을 때 |
| **Marp (마크다운)** | 마크다운으로 슬라이드 | 코드 중심 영상 |
| **PowerPoint** | 애니메이션 강력 | 단계별 빌드업 연출 |

### 화면 레이아웃

#### 기본: 슬라이드 + 아바타
```
┌──────────────────────────────────┐
│                           ┌─────┐│
│     PPT 슬라이드           │아바 ││
│     (메인 화면)            │타   ││
│                           └─────┘│
│  [하단] 현재 챕터 텍스트          │
└──────────────────────────────────┘
```

#### 코드 시연: 화면녹화 + 아바타
```
┌──────────────────────────────────┐
│                           ┌─────┐│
│     터미널 / IDE           │아바 ││
│     (화면녹화)             │타   ││
│                           └─────┘│
│  [하단] 실행 중인 명령어 요약      │
└──────────────────────────────────┘
```

#### 비교: 분할 화면
```
┌─────────────┬──────────────────┐
│             │                  │
│  Before     │     After        │
│  (문제)     │     (해결)       │
│             │                  │
└─────────────┴──────────────────┘
```

### 장면 전환 흐름도

```
타이틀(아바타 인사)
  ↓
목차 슬라이드
  ↓
문제 설명(PPT 다이어그램)
  ↓
시도 1(PPT → 화면녹화 → PPT 결과)  ←─┐
  ↓                                    │ 반복
시도 N(PPT → 화면녹화 → PPT 결과)  ───┘
  ↓
최종 해결(PPT 요약표 + 화면녹화 시연)
  ↓
정리(PPT 핵심 포인트)
  ↓
아웃트로(아바타 + 엔딩 슬라이드)
```

## 3. 촬영/편집 팁

### VTuber 아바타 세팅

#### 아바타 제작

| 도구 | 타입 | 비용 | 특징 |
|------|------|------|------|
| **VRoid Studio** | 3D | 무료 | 자유도 높음, 직접 모델링 |
| **Live2D + nizima** | 2D | 유료 | 고퀄리티 일러스트 느낌 |
| **Ready Player Me** | 3D | 무료 | 빠른 아바타 생성 |
| **외주 제작** | 2D/3D | 10-50만원 | 퀄리티 최고, 코코넛츠/크몽 |

**추천 (초반 빠른 시작):** VRoid Studio로 직접 만들기 or Ready Player Me

#### 아바타 트래킹 (움직임 연동)

| 도구 | 플랫폼 | 비용 | 입력 |
|------|--------|------|------|
| **VSeeFace** | Windows | 무료 | 웹캠 |
| **VTube Studio** | Win/Mac | 무료+유료 | 웹캠/iPhone |
| **Animaze (by FaceRig)** | Windows | 유료 | 웹캠 |
| **VMagicMirror** | Windows | 무료 | 웹캠, 키보드/마우스 연동 |

**추천:** VSeeFace (무료, 가볍고 OBS 연동 쉬움) or VTube Studio (iPhone 있으면 트래킹 퀄리티 높음)

#### OBS + 아바타 연동

```
OBS 씬 구성:
1. [슬라이드 + 아바타] - PPT 설명용
   - 소스: 윈도우캡처(슬라이드) + 아바타(VSeeFace 크로마키)
   
2. [화면녹화 + 아바타] - 코드/터미널 시연용
   - 소스: 화면캡처 + 아바타(PIP)
   
3. [아바타 풀] - 인트로/아웃트로용
   - 소스: 아바타만 큰 화면

아바타 크로마키 설정:
- VSeeFace에서 배경색 → 그린스크린
- OBS에서 크로마키 필터 적용
- 아바타 크기: 화면의 15-20% (PIP)
```

### 녹화 도구

| 도구 | 용도 | 비고 |
|------|------|------|
| **OBS Studio** (무료) | 화면녹화 + 아바타 합성 | 메인 녹화 도구 |
| **VSeeFace** (무료) | 아바타 트래킹 | 웹캠으로 표정/움직임 |
| **마이크** | 음성 | USB 콘덴서 (FIFINE, Blue Yeti 등) |
| **스마트폰** | 하드웨어 클로즈업 (필요시) | 거치대 필수 |

### OBS 기본 설정

```
녹화 설정:
- 해상도: 1920x1080
- FPS: 30
- 인코더: x264 or NVENC
- 비트레이트: 10,000-15,000 kbps
- 포맷: MKV (녹화 중 크래시 방지, 나중에 MP4로 리먹스)

씬 구성:
1. [아바타 풀] - 인트로/아웃트로용
2. [슬라이드 + 아바타 PIP] - 강의 설명용
3. [화면 + 아바타 PIP] - 코딩/터미널용
4. [화면 풀] - 코드 집중 구간용
```

### 녹화 팁

**슬라이드 녹화:**
- 슬라이드를 미리 PDF 또는 프레젠테이션 모드로 띄우기
- OBS에서 해당 창만 캡처 (전체 화면 아님)
- 슬라이드 넘길 때 잠깐 멈추고 설명 시작 (편집 포인트)
- 한 슬라이드당 30초-2분 (넘으면 슬라이드 쪼개기)

**터미널/코드 녹화:**
- 폰트 크기: 최소 16pt (시청자가 모바일에서도 읽을 수 있게)
- 터미널 테마: 어두운 배경 + 밝은 글자 (Dracula, One Dark 등)
- 중요한 명령어 입력 전 잠깐 멈추기
- 긴 출력은 핵심 부분만 화면에 보이게 스크롤

**하드웨어 (필요 시):**
- 스마트폰으로 탑다운/클로즈업 별도 촬영
- 편집 때 슬라이드 사이에 삽입
- 핀 번호, 커넥터가 보이도록 충분히 가까이

### 편집 워크플로우

**추천 편집 도구:**
- **DaVinci Resolve** (무료) - 전문적, 색보정 강력
- **CapCut** (무료) - 빠른 편집, 자막 자동생성
- **Premiere Pro** (유료) - 업계 표준

**편집 순서:**
1. 러프 컷: 불필요한 침묵, "음...", 반복 삭제
2. 구조 정리: 슬라이드 전환 타이밍 맞추기
3. 텍스트 오버레이: 핵심 키워드, 명령어, 에러 메시지 강조
4. 자막: CapCut 자동생성 → 수정 (가장 빠름)
5. 챕터 마커: YouTube 타임스탬프용 구간 메모
6. BGM: 잔잔한 로파이, 볼륨 낮게 (목소리 우선)
7. 인트로/아웃트로 그래픽: 채널 로고, 구독 버튼
8. 최종 확인: 소리 레벨, 화면 전환, 오탈자

**시간 절약 팁:**
- 블로그 글 먼저 쓰고 → 대본 뼈대로 변환 → PPT 제작 → 녹화
- 반복되는 인트로/아웃트로는 OBS 씬으로 템플릿화
- 실패 과정이 길면 타임랩스 (4-8배속) + 요약 슬라이드

## 5. 대본 → 번역 → 자막 워크플로우

### 전체 흐름

```
① 대본 작성 (KO)          docs/scripts/YYYY-MM-DD-[slug].md
      ↓
② 영어 번역 (EN)          같은 파일에 KO/EN 병렬 작성
      ↓
③ 녹화                    한국어 대본 보면서 녹화
      ↓
④ 타임코드 확정            편집하면서 각 구간 실제 시간 확인
      ↓
⑤ SRT 자막 생성            KO SRT + EN SRT 각각 생성
      ↓
⑥ YouTube 업로드           자막 파일 2개 첨부
```

### ① 대본 작성

대본 템플릿: `docs/templates/script-template.md`를 복사해서 사용.

```
docs/scripts/
  ├── 2026-04-10-zed-x-mini-jetson.md
  ├── 2026-04-17-stm32-rtos-intro.md
  └── ...
```

**작성 규칙:**
- 각 섹션에 `[PPT - 슬라이드 설명]` `[화면녹화]` 등 장면 지시 포함
- 실제로 말할 내용을 구어체로 작성 (문어체 X)
- 타임코드는 예상 시간으로 일단 작성, 편집 후 확정

### ② 영어 번역 (KO/EN 병렬)

대본 파일에 한국어와 영어를 **병렬로** 작성:

```markdown
## 인트로 (0:00 ~ 0:30)

**KO:**
안녕하세요, 로봇 엔지니어 헥세이치입니다.
오늘은 ZED X Mini를 Jetson에 연결하는 과정을 공유합니다.

**EN:**
Hi, I'm 0xH, a robotics engineer.
Today I'll share how I connected a ZED X Mini to a Jetson board.
```

**번역 규칙:**
- 직역보다 자연스러운 영어 (기술 용어는 그대로)
- 한국어 뉘앙스를 살리되, 영어권 시청자가 이해할 수 있게
- 문장 수를 비슷하게 맞추기 (자막 타이밍 동기화를 위해)

**예시 대본:** `docs/templates/script-example-zed.md` 참고

### ③~④ 녹화 & 타임코드 확정

녹화 후 편집하면서 실제 타임코드를 대본에 업데이트:

```markdown
## 인트로 (0:00 ~ 0:28)     ← 실제 시간으로 수정
## 목차 (0:28 ~ 0:55)
## 문제 설명 (0:55 ~ 2:48)
...
```

### ⑤ SRT 자막 생성

대본의 KO/EN 텍스트 + 확정된 타임코드로 SRT 파일 생성:

**한국어 자막 (ko.srt):**
```srt
1
00:00:00,000 --> 00:00:03,500
안녕하세요, 로봇 엔지니어 헥세이치입니다.

2
00:00:03,500 --> 00:00:07,000
오늘은 ZED X Mini를 Jetson에
연결하는 과정을 공유합니다.

3
00:00:07,000 --> 00:00:12,000
공식 문서에는 연결하고 SDK 설치하면
끝이라고 하는데, 저는 한 달 걸렸습니다.
```

**영어 자막 (en.srt):**
```srt
1
00:00:00,000 --> 00:00:03,500
Hi, I'm 0xH, a robotics engineer.

2
00:00:03,500 --> 00:00:07,000
Today I'll share how I connected
a ZED X Mini to a Jetson board.

3
00:00:07,000 --> 00:00:12,000
The official docs say just plug in
and install the SDK. It took me 30 days.
```

**SRT 작성 규칙:**
- 한 자막당 최대 2줄, 줄당 최대 42자 (한국어) / 42자 (영어)
- 한 자막 표시 시간: 1-7초
- 타임코드: `HH:MM:SS,mmm` 형식
- 문장이 길면 자연스러운 지점에서 나누기

**자막 파일 저장:**
```
docs/scripts/
  ├── 2026-04-10-zed-x-mini-jetson.md      (대본)
  ├── 2026-04-10-zed-x-mini-jetson.ko.srt  (한국어 자막)
  └── 2026-04-10-zed-x-mini-jetson.en.srt  (영어 자막)
```

### ⑥ YouTube 업로드 시 자막 첨부

```
YouTube Studio → 영상 → 자막 탭
  → "파일 업로드" → "자막 포함" 선택
  → ko.srt 업로드 (한국어, 기본)
  → en.srt 업로드 (영어)
```

### 자막 제작 시간 절약 팁

| 방법 | 설명 | 추천도 |
|------|------|--------|
| **대본 기반 직접 작성** | 대본이 있으니 타임코드만 맞추면 됨 | 가장 정확 |
| **CapCut 자동생성 → 수정** | 한국어 자동 인식 → 오타 수정 → SRT 내보내기 | 가장 빠름 |
| **Whisper (AI)** | 오픈소스 음성인식으로 자동 생성 | 무료+정확도 높음 |

**추천 워크플로우:**
1. 한국어 자막: CapCut 자동생성 → 대본 대조하며 수정 → SRT 내보내기
2. 영어 자막: 대본의 EN 텍스트 + 한국어 자막의 타임코드 → EN SRT 생성

  □ 썸네일 제작
  □ 1080p MP4 내보내기

업로드
